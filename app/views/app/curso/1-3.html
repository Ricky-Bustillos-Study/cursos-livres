Gerenciamento de Pacotes

O que é?

Um gerenciador de pacotes é uma ferramenta utilizada para instalação, remoção e atualização de programas. Normalmente são encontrados em sistemas GNU/Linux, tendo em vista que tais sistemas consistem de milhares de pacotes distintos.

    Debian Source
  • .deb-src
    • Debian
  • .deb

Distribuições Linux Debian

O Debian possui diversas distribuições de pacotes, cada uma delas possui três principais componentes:

    Componente I
  • contrib

    • Onde possui todas as contribuições do pacote.

    Componente II
  • main

    • Onde possui os principais arquivos da distribuição.

    Componente III
  • non-free

    • Pacotes e softwares não-livres.

Como configurar o Linux para Instalações APT

É necessário instalar e configurar alguns pacotes no sistema do Linux.


    Habilitando e adicionando diretórios de recursos

  1. $ cd /etc/apt



  2. $ sudo nano sources.list



  3. Comentar tudo o que houver neste arquivo utilizando o caracter #
  4. Digitar os seguintes diretórios abaixo dos textos comentados do arquivo:
    • deb http://ftp.br.debian.org/debian/ stable main contrib non-free
    • deb-src http://ftp.br.debian.org/debian/ stable main contrib non-free
    • deb http://ftp.br.debian.org/debian/ stable-updates main contrib non-free
    • deb-src http://ftp.br.debian.org/debian/ stable-updates main contrib non-free
    • deb http://security.debian.org/ stable/updates main
    • deb-src http://security.debian.org/ stable/updates main
    • deb http://ftp.debian.org/debian stretch-backports main
    • deb-src http://ftp.debian.org/debian stretch-backports main
    Pressionar Ctrl + X para sair do arquivo. Logo então ele pedirá para salvar o arquivo, então pressione S e em seguida pressione Enter duas vezes.

    5. Configurando a placa de rede


  5. $ cd /etc/network



  6. $ sudo nano interfaces



  7. Apagar tudo.
  8. Digitar a configuração(interfaces) de rede em modo bridge.
      NAT
    • auto lo
    • iface lo inet loopback
    • auto enp0s3
    • iface enp0s3 inet dhcp

      Bridge
    • auto lo
    • iface lo inet loopback
    • auto enp0s3
    • iface enp0s3 inet static
    • address 192.168.15.(numero-da-maquina)
    • netmask 255.255.255.0
    • gateway 192.168.15.254
    • broadcast 192.168.15.255

    9. Configurando DNS


  9. $ cd /etc



  10. $ sudo nano resolv.conf



  11. Apagar tudo o que há inserido neste arquivo e incluir:
    • nameserver 8.8.8.8
    • nameserver 8.8.4.4

    12. Reiniciar o serviço de rede


  12. $ sudo etc/init.d/networking restart



    13. Atualizando o APT


  13. $ sudo apt update



Comando APT e APT-GET

O comando apt é uma melhoria do comando apt-get, possuindo as mesmas funções e algumas possibilidades a mais.

O apt é uma ferramenta para manipulação de pacotes no Linux, sendo controlada apenas pelo usuário root.

$ sudo apt <instrução>



Instalar pacotes

Instalar um pacote ao sistema operacional é dado pelo comando:

$ sudo apt install <nome-do-pacote>



Purgar e Remover pacotes

Remover um pacote ou desinstalar todos os arquivos relacionados com o pacote pode ser dado pelo comando (não apaga as configurações):

$ sudo apt remove <nome-do-pacote>



Purgar um pacote significa desinstalar todos os arquivos bináros relacionados co o pacote e todos os arquivos de configuração, é dado pelo comando:

$ sudo apt purge <nome-do-pacote>

Atualizar e Melhorar pacotes

A atualização é dada pela instrução update do comando apt, logo é dado:

$ sudo apt update



A melhora é dada pela instrução upgrade do comando apt, logo é dado:

$ sudo apt upgrade;

Cron

O cron é um agendador de tarefas. Possui alguns atributos:

  • $ sudo crontab -l



    • Ele lista o conjunto de argumentos que está presente no sistema.

  • $ sudo crontab -e



    • Ele edita a lista de argumentos do cron.
