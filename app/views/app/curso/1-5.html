Processo

Um processo é um programa em execução, todo processo possui um PID (Process Identification), que é o identificador do processo, em algarismos.

Um processamento são cálculos executados na CPU, na qual é denominado CPU Bound, ou seja, quando um processo está em execução no processador.

Quando um processo é I/O Bound ele aguarda uma aquisição ou uma saída de dados.

Processo Zumbi

Um processo zumbi é um processo na qual não possui um PID. Ele é um processo comum na qual perdeu seu processo pai, na qual consequentemente, acarreta na perda de seu PID.

Gerenciamento

Quando executamos algum comando, script ou iniciamos alguma programa, o kernel atribui a ele um número de processo (PID) e passa a gerenciar a quantidade de recursos que ele irá disponibilizar para essa atividade.

Como haverá sempre diversos processos rodando simultaneamente na máquina o kernel tem uma lista de processos que necessitam de recursos. Como não existe atualmente um sistema realmente multitarefa, capaz de realizar diversas atividades realmente ao mesmo tempo, o kernel cria uma fila de processos e a percorre disponibilizando recursos de máquina para cada um deles por um determinado tempo. Quanto melhor essa distribuição for efetuada, melhor será o desempenho do sistema como um todo e mais próximo de um sistema multitarefas o sistema se parecerá.

A forma como o kernel gerencia os processos é bastante inteligente e podemos sempre visualizar o status do processo num determinado instante, para determinar se ele está sendo executado neste mesmo instante ou se ele está aguardando tempo de máquina para que seja executado. Podemos visualizar todos os processos que estão rodando em nosso sistema com o programa que tira uma foto dessa estrutura, conhecido como snaPShot vulgo ps.

Threads

Threads são processos que apresentam memória compartilhada.

MultiThread

São várias threads em execução compartilhando um mesmo alocamento.

Comandos KILL, XKILL e KILLALL

Esses comandos tem como função matar um processo (fechar um processo) que esteja em execução no sistema.

  • Kill

    • Finaliza o processo a partir do PID indicado no comando:

    $ sudo kill <PID>



    Finaliza o processo:

    $ sudo kill -9 <PID>



    Finaliza o processo em espera:

    $ sudo kill -19 <PID>



    Finaliza o processo em execução:

    $ sudo kill -18 <PID>



  • XKill

    • É uma interface para finalizar processos.

  • KillAll

    • Finaliza todos os processos com o mesmo nome.

    $ sudo killall <nome-processo>



Comandos PGREP e PIDOF

Os comandos pgrep e pidof determinam o número de identificação do processo (PID) por meio da busca do nome informado. É utilizado da seguinte forma:

$ pgrep <nome-processo>



ou ...

$ pidof <nome-processo>



Comando JOBS

Ele apresenta o estado dos processos executados pelo user/root em seu respectivo terminal (TTY).

$ jobs



Daemon

Um arquivo Daemon é um programa que sempre está sendo executado pelo sistema, é um processo que nunca morre, a não ser quando a máquina é desligada.

Colunas do PS AUX

  • USER

    • Nome do usuário dono do processo.

  • UID

    • Número de identificação do usuário dono do processo.

  • PID

    • Número de identificação do processo.

  • PPID

    • Número de identificação do processo pai.

  • \% CPU

    • Porcentagem do processamento usado.

  • \% MEM

    • Porcentagem da memória usada.

  • VSZ

    • Indica o tamanho virtual do processo.

  • RSS

    • Resident Set Size, indica a quantidade de memória usada (em Kbytes).

  • TTY

    • Indica o identificador do terminal do processo.

  • START

    • Hora em que o processo foi iniciado.

  • COMMAND

    • Nome do comando que executa aquele processo.

  • PRI

    • Valor da prioridade do processo.

  • NI

    • Valor preciso da prioridade (geralmente igual aos de PRI).

  • WCHAN

    • Mostra a função do kernel ondeo processo se encontra em modo suspenso.

  • STAT

    • Indica o estado atual do processo, sebdi representado por uma letra.

Comando TOP

O Comando top funciona como um gerenciador de tarefas/processos no sistema.

$ top



Comando LSOF

Esse comando é importante para o sistema devido sua capacidade de listar todos os arquivos abertos por todos os processos. Quando falamos em arquivos isso significa que podem ser bibliotecas, sockets, arquivos comuns, diretórios, etc.

Assim, o comando fornece um mapeamento do que o programa está usando no sistema.

$ lsof
